## A.2 Technical Documentation

### A.2.1 System Architecture Overview

VEBStore implements a modern three-tier architecture with clear separation of concerns:

**Frontend Layer (React.js):**
- **Technology Stack**: React 18, Vite, Tailwind CSS, React Router
- **Component Architecture**: Modular component-based design with reusable UI elements
- **State Management**: React Context API with custom hooks for global state
- **Performance Optimization**: Code splitting, lazy loading, and caching strategies
- **Key Components**: 
  - ProductList.jsx - Product display and filtering
  - Cart.jsx - Shopping cart management
  - Checkout.jsx - Order processing
  - AdminPanel.jsx - Administrative interface

**Backend Layer (Node.js):**
- **Technology Stack**: Node.js 18 LTS, Express.js, MongoDB, Mongoose ODM
- **API Design**: RESTful API with comprehensive endpoint documentation
- **Middleware Stack**: Authentication, validation, error handling, and logging
- **Security Implementation**: JWT authentication, data encryption, and rate limiting
- **Key API Endpoints**:
  - `/api/auth/*` - Authentication and user management
  - `/api/product/*` - Product catalog management
  - `/api/order/*` - Order processing and management
  - `/api/payment/*` - Payment integration with Razorpay

**Database Layer (MongoDB):**
- **Database System**: MongoDB with document-based storage
- **Data Models**: User, Product, Order, and Review schemas with proper relationships
- **Indexing Strategy**: Optimized indexes for query performance
- **Data Integrity**: Validation rules and constraints at database level
- **Schema Definitions**:
  - User Schema: Authentication, profile, cart data
  - Product Schema: Product details, inventory, reviews
  - Order Schema: Order processing, status, tracking
  - Review Schema: Customer feedback and ratings

### A.2.4 Integration Documentation

**Razorpay Payment Gateway Integration:**

1. **Configuration Setup:**
   ```javascript
   const Razorpay = require('razorpay');
   const razorpay = new Razorpay({
     key_id: process.env.RAZORPAY_KEY_ID,
     key_secret: process.env.RAZORPAY_KEY_SECRET
   });
   ```

2. **Payment Flow:**
   - User selects products and proceeds to checkout
   - Backend creates Razorpay order with amount and currency
   - Frontend receives order ID and opens Razorpay checkout modal
   - User completes payment through Razorpay interface
   - Razorpay sends payment confirmation to webhook endpoint
   - Backend verifies payment and updates order status

3. **Webhook Handling:**
   ```javascript
   app.post('/api/payment/webhook', (req, res) => {
     const crypto = require('crypto');
     const signature = req.headers['x-razorpay-signature'];
     const expectedSignature = crypto
       .createHmac('sha256', process.env.RAZORPAY_WEBHOOK_SECRET)
       .update(JSON.stringify(req.body))
       .digest('hex');
     
     if (signature === expectedSignature) {
       // Process payment confirmation
       // Update order status
       // Send confirmation email
     }
   });
   ```

- **State Management**: React Context API with custom hooks for global state
- **Performance Optimization**: Code splitting, lazy loading, and caching strategies

**Backend Layer:**
- **Technology Stack**: Node.js, Express.js, MongoDB, Mongoose ODM
- **API Design**: RESTful API with comprehensive endpoint documentation
- **Middleware Stack**: Authentication, validation, error handling, and logging
- **Security Implementation**: JWT authentication, data encryption, and rate limiting

**Database Layer:**
- **Database System**: MongoDB with document-based storage
- **Data Models**: User, Product, Order, and Review schemas with proper relationships
- **Indexing Strategy**: Optimized indexes for query performance
- **Data Integrity**: Validation rules and constraints at database level
